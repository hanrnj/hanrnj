exe -> 윈도우용 os 실행파일 
하드디스크(보조기억장치 hdd)에 응용프로그램(exe) 파일이 저장되어있다.

<일반적인 프로그램 실행시>
exe를 실행시켰을때 주체는 os 
exe파일을 실행시켰을때 os가 하는일 첫번째는 하드디스크(보조기억장치)에서 메모리(주기억장치)로 프로그램파일을 올린다.
그 후에 cpu로 전달
os는 역할 두번째 메모리에서 프로그램의 공간(주소) 관리

자바 실행법
소스코드를 만들어서 소스코드를 컴파일해서 클레스 파일(바이트 코드)로 만들고나서(   ).class파일을 실행한다.

컴파일러란 씨피유(자바에서는 JVM)가 알아들을 수 있는 2진수 언어로 바꾸는 것 (자바에서는).class파일로 만드는 것

컴파일이란 내가만든 소스코드를 컴퓨터가 이해하는 2진수로 바꿔서(.Class) 파일로 만들어라

실행은 자바  ().class 

함수 = 메소드, 자바 소스는 클레스로 시작해서 클레스 안에는 함수가 올 수 있다.
객체지향언어는 클레스 밖에는 함수가 존재할 수 없다.
클레스명 첫번째는 무조건 대문자!!!


("" = 문자열 )  앞에 문자열 "" 이 있으면 뒤에도 문자취급
/*(   ) */ 주석안에 있는 것들은 컴파일러가 클레스 파일로 변환하지 않는다

컨트롤+쉬프트+/ : 블록을 주석으로 처리
컨트롤+쉬프트+백스페이스 : 블록 주석을 해제
컨트롤+/ 여러 줄을 주석처리/헤제

{ int(type,자료형) num;(변수명) } -> 변수(값이 변할 수 있는) 선언(메모리에 정수가 들어갈 수 있는 num이라는 이름의 4byte(32bit) 공간을 할당해라=메모리 할당)
num =(대입) 10; (메모리에 num이라는 4byte 공간에 10이라는 10진수를 2진수 0x28 1010으로 대입한다) 

메모리에는 주소가있다
os가 메모리에 프로그램코드를 올리는데 올리는 주소를 결정하는것은 os가 한다.
